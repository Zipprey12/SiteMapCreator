# SiteMapCreator

Консольное приложение для построения карты сайта. Обходит страницы сайта по ссылкам, строит дерево URL и сохраняет результат в текстовый файл.

## Возможности

- Рекурсивный обход страниц сайта по тегам `<a href>`
- Многопоточная обработка на базе `ForkJoinPool` (`RecursiveTask`)
- Два режима парсинга: с учётом query-параметров (`?param=value`) и без
- Настраиваемая глубина обхода, лимит страниц и задержка между запросами
- Обработка ошибок соединения с подсчётом неудачных запросов
- Сохранение карты сайта в файл с отображением вложенности

## Технологии

- Java 21
- [Jsoup](https://jsoup.org/) — парсинг HTML и HTTP-запросы
- [Lombok](https://projectlombok.org/) — снижение boilerplate-кода
- Maven

## Запуск

### Через IDE

Открыть проект как Maven-проект и запустить класс `Main`.

### Через Maven

```bash
mvn compile exec:java -Dexec.mainClass=mapper.Main
```

## Использование

После запуска приложение предложит меню:

```
1 - Лимит вложенности поиска
2 - Максимальное количество страниц
3 - Задержка между запросами
4 - Задать режим парсинга сайта
5 - Создать карту сайта
6 - Выход
```

Порядок работы:

1. Настроить параметры при необходимости (по умолчанию: глубина 3, лимит 3000 страниц, задержка 150 мс, режим EXTENDED)
2. Выбрать «Создать карту сайта» и ввести адрес, например `https://example.com`
3. Готовый файл сохранится в папку `map_files/`

### Режимы парсинга

`EXTENDED` — обходит все уникальные URL, включая query-параметры (`/search?q=java` и `/search?q=python` считаются разными страницами).

`FAST` — отбрасывает query-параметры, обходит только уникальные пути. Быстрее, меньше запросов.

```
src/main/java/mapper/
  Main.java
  model/                      # Доменные модели
    Link.java                 # Ссылка и её составные части

    MapFillerArgs.java        # Аргументы для многопоточного обхода

  repository/
    output/MapWriter.java     # Запись карты в файл
    parsers/                  # Парсинг HTML-страниц
  services/
    Navigator.java            # Консольное меню
    commands/                 # Команды меню
    links/                    # Фабрики для работы со ссылками
    map/                      # Построение карты сайта
      filler/                 # Многопоточный обход (ForkJoinPool)
```

## Ключевые компоненты

**`SiteMapFiller`** — рекурсивная задача (`RecursiveTask`), обходит страницы в `ForkJoinPool`. Каждая страница превращается в отдельную задачу. Раздельно обрабатывает `SocketException`, `SocketTimeoutException` и `HttpStatusException`, ведёт счётчики обработанных и неудачных запросов.

**`ILinksFactory` / `RelativeLinksFactory` / `FastLinksFactory`** — иерархия фабрик для создания ссылок из сырых `href`. `FastLinksFactory` реализует паттерн декоратор: оборачивает базовую фабрику и отбрасывает query-параметры до передачи URL в `createLink`.

**`LinksFactory`** — абстрактный базовый класс, инкапсулирует парсинг входного URL: извлекает протокол, домен и путь, строит `mainPage` и `initialPage`. Позволяет задавать протокол отдельно — для случаев, когда пользователь вводит URL без схемы.

**`MapFillersFactory`** — точка сборки: хранит настройки обхода (глубина, лимит страниц, задержка, режим парсинга) и управляет переключением между режимами через `LinksParser.setParsingType`.

**`PageNavigationLinksParser`** — парсит HTML-страницу через Jsoup, фильтрует изображения и передаёт найденные `href` в активную `ILinksFactory`. Поддерживает замену фабрики на лету при смене режима парсинга.
